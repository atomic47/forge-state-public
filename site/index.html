§ 02 · Step 1 — Install the plugin

Paste two lines into Claude.

From the marketplace · recommended 
› /plugin marketplace add atomic-47-labs/forge-state-public
› /plugin install forge-state@forge-state
Or download the tarball
github.com/atomic-47-labs/forge-state-public/releases Then /plugin marketplace add ~/Downloads/forge-state

Installs 14 /forge-* skills under a single plugin.

§ 03 · Step 2 — Install Docker + GitHub CLI

Two tools forge needs on the host.

Sandbox · the data plane
Docker Desktop

Every experiment clones, builds, and runs inside a fresh container. No host shell exposure.

docker.com/products/docker-desktop
macOS · Windows · Linux
Intake · how forge talks to GitHub
GitHub CLI

forge clones repos and inspects releases through gh — no PAT in the shell.

cli.github.com
After install: gh auth login

Both are one-time installs. Forge checks them automatically on first run.

✓

§ 05 · Step 4 — Run forge

In the Code tab, open a folder and go.

Create a working directory · open it · run 
$ mkdir ~/forge-orchestrate && cd ~/forge-orchestrate
$ claude   # or open the folder in the Claude Desktop Code tab

› /forge-orchestrator
FIRST RUN
Creates ~/forge/ — manifest, state, experiments tree.
EVERY RUN
Harvest · research · build · experiment · package · report · publish.

Mark a repo with 🧪 in your configured Slack channel — or paste a GitHub URL into /forge-harvester-slack directly. Forge does the rest.
